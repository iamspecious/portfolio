Cat API — GET /v1/images/search

Specious Coda-Bishop · Technical Writing

Reference documentation for The Cat API's images search endpoint. Covers authentication, query parameters, response schema, error handling, and pagination behaviour — written to production documentation standards.

Portfolio Context

This document is a technical writing sample produced for a JetBrains application. The Cat API is a real, publicly available REST API. All parameters, response fields, error codes, and behaviour documented here reflect the actual API — this is not a fictional example.

The goal was to demonstrate API documentation to a production standard: precise parameter tables, accurate authenticated vs. unauthenticated distinctions, copy-pasteable examples, and error states written for resolution rather than enumeration.

01
Overview

The /v1/images/search endpoint returns a list of cat images matching the specified filters. It is the primary way to retrieve image data from The Cat API — whether that's a single random image for a quick embed, a filtered set of images for a specific breed, or a paginated result set for a browsable gallery.

Unauthenticated requests return one random image with no filtering or pagination. An API key unlocks batch retrieval up to 25 results per request, breed and category filtering, and stable paginated result sets.

Use caseApproach
Display a random cat imageCall without an API key — returns one random image by default
Retrieve images for a specific breedUse breed_ids with a valid API key
Build a browsable galleryUse order=ASC with page and limit to paginate through a stable result set
Filter by file typeUse mime_types to restrict to jpg, png, or gif
Embed a raw image directlySet format=src — the API responds with the image file rather than a JSON body
02
Authentication

The API uses key-based authentication passed as a request header.

HeaderValue
x-api-keyYour API key from the Cat API dashboard

Get a free API key at thecatapi.com. You can make unauthenticated requests, but they are significantly restricted.

UnauthenticatedAuthenticated
Max results per request125
Filter by breed or categoryNoYes
PaginationNoYes
Monthly request limit—10,000 (free) / 100,000 (premium)
Note

The header name is lowercase: x-api-key. Using a different casing (e.g. X-Api-Key) may cause the key to be ignored in some environments, returning a 401.

03
Base URL and Method
GET https://api.thecatapi.com/v1/images/search

All requests to this endpoint use the HTTP GET method. Parameters are passed as query strings. No request body is required or accepted.

04
Query Parameters All parameters are optional
ParameterTypeDefaultDescriptionExample
limit integer 1 Number of results to return. Maximum 25. Values above 1 require a valid API key. limit=10
page integer 0 Page offset for paginated results. Zero-indexed. Only applies when order is ASC or DESC — ignored when order=RANDOM. page=2
order string RANDOM Sort order for results. RANDOM disables pagination. Accepted values: RANDOM, ASC, DESC. order=ASC
breed_ids string — Comma-separated breed IDs to filter by. Returns only images associated with the specified breeds. See GET /v1/breeds for valid IDs. breed_ids=beng,abys
category_ids string — Comma-separated category IDs to filter by. See GET /v1/categories for valid IDs. category_ids=1,5
has_breeds integer 0 When set to 1, returns only images that have breed data attached. Useful when you need breed information guaranteed in the response. has_breeds=1
mime_types string All types Comma-separated list of image types to include. Accepted values: jpg, png, gif. mime_types=jpg,png
size string med Image size variant to return. Accepted values: thumb, small, med, full. size=full
format string json Response format. json returns a JSON array. src returns the raw image file directly — useful for embedding in an <img> tag. format=json
Heads Up — breed_ids and category_ids require an API key

Filtering by breed_ids or category_ids requires a valid x-api-key header. These parameters are silently ignored on unauthenticated requests rather than returning an error.

05
Request Example

Retrieve 5 Bengal cat images in ascending order, with breed data included, starting from the first page.

curl -X GET "https://api.thecatapi.com/v1/images/search?limit=5&has_breeds=1&breed_ids=beng&order=ASC&page=0" \
  -H "x-api-key: YOUR_API_KEY"

Minimal unauthenticated request

Returns one random image with no breed data. No headers required.

curl -X GET "https://api.thecatapi.com/v1/images/search"

Response Schema

The endpoint returns a JSON array. Each element is an image object.

Image Object
FieldTypeDescription
idstringUnique identifier for the image.
urlstringDirect URL to the image file.
widthintegerImage width in pixels.
heightintegerImage height in pixels.
breedsarrayBreed objects associated with this image. Empty array if no breeds are attached. Populated when has_breeds=1 or breed_ids is specified.
categoriesarrayCategory objects associated with this image. Empty array if no categories match.
Breed Object (within breeds)
FieldTypeDescription
idstringUnique breed identifier (e.g. beng for Bengal).
namestringBreed name.
descriptionstringSummary description of the breed.
temperamentstringComma-separated temperament traits.
originstringCountry of origin.
life_spanstringTypical lifespan range in years (e.g. "12 - 15").
wikipedia_urlstringWikipedia URL for this breed.
reference_image_idstringID of the canonical reference image for this breed.
Pagination Response Header

When order is ASC or DESC, the response includes an additional header:

HeaderTypeDescription
pagination-countintegerTotal number of results matching the current query. Use this to calculate total pages: ceil(pagination-count / limit).
Note

pagination-count is absent when order=RANDOM. Random requests return an independent set each time — there is no stable total to paginate against.

08
Error States
StatusMeaningCauseResolution
400 Bad Request A query parameter contains an unrecognised value (e.g. order=NEWEST). Check parameter names and accepted values against the table in Section 04.
401 Unauthorized The x-api-key header is missing or contains an invalid key. Confirm the header name is lowercase (x-api-key) and the value matches your dashboard key exactly. If using an environment variable, check it is being interpolated correctly.
429 Too Many Requests Monthly request allowance has been exhausted. Upgrade to a premium plan, or wait for the allowance to reset at the start of the next billing month.
500 Internal Server Error An unexpected error on the API side unrelated to your request. Retry with exponential backoff. If the issue persists, check thecatapi.com for service status.
09
Notes
Pagination Behaviour

Pagination is only available when order is ASC or DESC. When order=RANDOM (the default), the page parameter is ignored and each request returns an independent random set — there is no stable sequence to page through.

When pagination is active, check the pagination-count response header for the total number of results matching your query. Use it to calculate total pages:

total_pages = ceil(pagination-count / limit)
Rate Limits

Free-tier accounts are limited to 10,000 requests per month. Premium accounts receive 100,000 requests per month. No per-second rate limit is documented. For high-volume usage, implement standard exponential backoff on 429 responses.

Authenticated vs. Unauthenticated Requests

Unauthenticated requests (no x-api-key header) always return exactly one random image, regardless of the limit or page values passed. The breed_ids, category_ids, and has_breeds parameters are silently ignored — they do not return an error, they simply have no effect.

Key Point

If filtering appears to have no effect on your request, confirm the x-api-key header is present and valid. Silent ignore is the failure mode for unauthenticated filter parameters — not an error response.

Using format=src

When format=src is set, the API responds with the raw image file rather than a JSON body. This allows you to use the endpoint URL directly as an image source:

<!-- Embeds a random cat image directly, no JavaScript required -->
<img src="https://api.thecatapi.com/v1/images/search?format=src&mime_types=jpg" alt="A random cat">

All other filtering parameters still apply when using format=src.
